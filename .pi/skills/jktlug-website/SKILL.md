# JKTLUG Website Skill

## Overview

The JKTLUG website (`jktlug.id`) is a static site generated by a
custom Hakyll compiler written in Haskell. It was forked from the
Tokyo Linux Users Group (`tlug/tlug.jp`) site and is being migrated
to Jakarta/Indonesia branding and content.

It serves two kinds of content:

1. **Static pages** (`docroot/`) — hand-written HTML/CSS/images
2. **Wiki pages** (`wiki/`) — MediaWiki markup, parsed by custom code into HTML

## Architecture

```
Source Files  →  Hakyll Compiler  →  _site/  →  Netlify CDN
(docroot/        (Haskell program)    (HTML)     (serves to users)
 wiki/
 template/)
```

Key modules:
- `app/SiteCompiler.hs` — Main Hakyll rules (routing + compilers)
- `src/JKTLUG/MediaWiki.hs` — Wiki markup parser + transclusion
- `src/JKTLUG/WikiLink.hs` — Link/image fixup after rendering
- `src/JKTLUG/WikiParam.hs` — Template parameter substitution
- `src/JKTLUG/Parser.hs` — Core parser combinator framework
- `template/main.html` — Site-wide page wrapper

## Build Commands

| Task | Command |
|---|---|
| Build + test | `./Test` |
| Compile site | `stack exec site-compiler build` |
| Full rebuild | `stack exec site-compiler rebuild` |
| Preview server | `stack exec site-compiler watch -- --host 0.0.0.0` |
| Tests only | `./Test --test-only` |
| Clean build | `./Test -C` |

**Locale fix:** If `htfpp` crashes with "cannot decode byte sequence",
ensure `LANG=C.utf8`. The Dockerfile has `ENV LANG=C.UTF-8`.

## Page Types and How They Render

### Static Pages (docroot/)

Files like `docroot/index.html` go through:

```haskell
match "docroot/*.html" $ do
    route   dropInitialComponent
    compile $ pandocCompiler
        >>= loadAndApplyTemplate "template/main.html" defaultContext
```

They are wrapped in `template/main.html` at the `$body$` placeholder.
Static assets (`docroot/css/`, `docroot/images/`) are copied verbatim.

### Wiki Pages (wiki/)

Files with no extension (e.g., `wiki/Meetings:2024:03`) go through
`mediawikiCompiler`, a multi-stage pipeline:

```
wiki/Some_Page
    │
    ▼
JKTLUG.MediaWiki.parseFile  ──► Parse into [Chunk]
    │                           Resolve {{transclusions}}
    │                           Substitute {{{parameters}}}
    │                           Handle <noinclude> / #REDIRECT
    ▼
Pandoc.readMediaWiki      ──► MediaWiki string → Pandoc AST
    ▼
Pandoc.writeHtml          ──► Pandoc AST → HTML string
    ▼
JKTLUG.WikiLink.fixWikiLinks ──► Fix [[links]], resolve images,
                                   extract categories, mark broken links
    ▼
loadAndApplyTemplate      ──► Wrap in template/main.html
    ▼
_site/wiki/Some_Page
```

### Category Pages

`[[Category:Foo]]` tags are extracted and auto-listed. The compiler
generates category listing pages from them.

## File Layout

| Directory | Purpose |
|---|---|
| `app/` | Entry points: `SiteCompiler.hs`, `Test.hs` |
| `src/JKTLUG/` | Parser library modules |

## Migration Context: TLUG → JKTLUG

This repo is actively being migrated. Common issues you will see:
- Old TLUG/Tokyo branding in HTML, templates, images
- Historical TLUG meeting pages in `wiki/`
- `tlug` references in URLs, mailing lists, external links
- Module names may still be `TLUG.*` during migration (should become `JKTLUG.*`)

When editing, always check: **is this content still Tokyo/TLUG specific?**
If so, replace with Jakarta/Indonesia equivalents.

## Common Tasks

### Fix content on the home page

Edit `docroot/index.html`. Run `stack exec site-compiler rebuild`.
No Haskell changes needed.

### Add or edit a wiki page

Create or edit files in `wiki/` using MediaWiki syntax:

| Syntax | Meaning |
|---|---|
| `[[Page Name]]` | Internal wiki link |
| `[[Page Name\|display text]]` | Link with custom text |
| `{{Template:Foo\|arg1\|arg2}}` | Transclude another page |
| `{{{1}}}` | Positional parameter in a template |
| `{{{name}}}` | Named parameter in a template |
| `[[Category:Foo]]` | Category tag (extracted for listing pages) |
| `#REDIRECT [[Target]]` | Page redirect |
| `<noinclude>...</noinclude>` | Content hidden when transcluded |

### Rename TLUG → JKTLUG module

This is a mechanical refactor across multiple files:
1. Rename directory `src/TLUG/` → `src/JKTLUG/`
2. Update `module TLUG.X` → `module JKTLUG.X` in each file
3. Update `import TLUG.X` → `import JKTLUG.X` everywhere
4. Update `tlug-website.cabal` module list

### Fix a broken wiki link

Broken links render in red (`<font color=#CC2200>`). Check:
1. Target file exists in `wiki/` (case-insensitive match).
2. Link syntax is correct.
3. If transcluded template, check `JKTLUG.MediaWiki.transFileName`.

### Change site layout

Edit `template/main.html`. `$body$` is the placeholder.
CSS is in `docroot/css/jktlug.css` (or `tlug4.css` during migration).

### Audit remaining TLUG references

```bash
grep -ri "tlug\|tokyo" --include="*.md" --include="*.html" \
  --include="*.hs" --include="*.cabal" --include="*.toml" \
  --include="*.yaml" --include="*.sh" \
  --exclude-dir=".pi" --exclude-dir="_site" \
  --exclude-dir=".stack-work" --exclude-dir=".git"
```

## Important Constraints

- **No server-side runtime.** Everything pre-generated. Interactivity
  needs client-side JS.
- **UTF-8 required.** Build fails without UTF-8 locale.
- **Transclusion recursion.** Parser resolves templates recursively.
  Avoid loops.
- **Wiki filenames use colons.** Valid on Linux, problematic on some
  filesystems.
- **Migration in progress.** TLUG content and branding are being
  replaced. Old references linger.

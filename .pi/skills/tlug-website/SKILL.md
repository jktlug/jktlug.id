# TLUG Website Skill

## Overview

The TLUG website (`tlug.jp`) is a static site generated by a custom
Hakyll compiler written in Haskell. It serves two kinds of content:

1. **Static pages** (`docroot/`) — hand-written HTML/CSS/images
2. **Wiki pages** (`wiki/`) — MediaWiki markup from the old site,
   parsed by custom code into HTML

## Architecture

```
Source Files  →  Hakyll Compiler  →  _site/  →  Netlify CDN
(docroot/        (Haskell program)    (HTML)     (serves to users)
 wiki/
 template/)
```

Key modules:
- `app/SiteCompiler.hs` — Main Hakyll rules (routing + compilers)
- `src/TLUG/MediaWiki.hs` — Wiki markup parser + transclusion
- `src/TLUG/WikiLink.hs` — Link/image fixup after rendering
- `src/TLUG/WikiParam.hs` — Template parameter substitution
- `src/TLUG/Parser.hs` — Core parser combinator framework (Functor/Applicative)

## Build Commands

| Task | Command |
|---|---|
| Build + test | `./Test` |
| Compile site | `stack exec site-compiler build` |
| Full rebuild | `stack exec site-compiler rebuild` |
| Preview server | `stack exec site-compiler watch -- --host 0.0.0.0` |
| Tests only | `./Test --test-only` |
| Clean build | `./Test -C` |

**Locale fix:** If `htfpp` crashes with "cannot decode byte sequence",
your shell needs `LANG=C.utf8`. The Dockerfile has `ENV LANG=C.UTF-8`;
if working outside the devcontainer, export it manually.

## Page Types and How They Render

### Static Pages (docroot/)

Files like `docroot/index.html` and `docroot/index.ja.html` go through:

```haskell
match "docroot/*.html" $ do
    route   dropInitialComponent
    compile $ pandocCompiler
        >>= loadAndApplyTemplate "template/main.html" defaultContext
```

They are wrapped in `template/main.html` at the `$body$` placeholder.
Static assets (`docroot/css/`, `docroot/images/`) are copied verbatim
via `copyFileCompiler`.

### Wiki Pages (wiki/)

Files with no extension (e.g., `wiki/Meetings:2024:03`) go through
`mediawikiCompiler`, a multi-stage pipeline:

```
wiki/Some_Page
    │
    ▼
TLUG.MediaWiki.parseFile  ──► Parse into [Chunk]
    │                           Resolve {{transclusions}}
    │                           Substitute {{{parameters}}}
    │                           Handle <noinclude> / #REDIRECT
    ▼
Pandoc.readMediaWiki      ──► MediaWiki string → Pandoc AST
    ▼
Pandoc.writeHtml          ──► Pandoc AST → HTML string
    ▼
TLUG.WikiLink.fixWikiLinks ──► Fix [[links]], resolve images,
                                extract categories, mark broken links
    ▼
loadAndApplyTemplate      ──► Wrap in template/main.html
    ▼
_site/wiki/Some_Page
```

### Category Pages

`[[Category:Foo]]` tags are extracted and auto-listed. The compiler
generates category listing pages from them.

| Directory | Purpose |
|---|---|
| `app/` | Entry points: `SiteCompiler.hs`, `Test.hs` |
| `src/TLUG/` | Parser library modules |
| `docroot/` | Static HTML/CSS/images. Copied to `_site/` |

| `_cache/` | Hakyll's internal cache for incremental builds |
| `doc/` | Developer documentation |

## Common Tasks

### Fix content on the home page

Edit `docroot/index.html` (English) or `docroot/index.ja.html`
(Japanese). Run `stack exec site-compiler rebuild`. No Haskell
changes needed.

### Add or edit a wiki page

Create or edit files in `wiki/` using MediaWiki syntax. Key syntax:

| Syntax | Meaning |
|---|---|
| `[[Page Name]]` | Internal wiki link |
| `[[Page Name\|display text]]` | Link with custom text |
| `{{Template:Foo\|arg1\|arg2}}` | Transclude another page |
| `{{{1}}}` | Positional parameter in a template |
| `{{{name}}}` | Named parameter in a template |
| `[[Category:Foo]]` | Category tag (extracted for listing pages) |
| `#REDIRECT [[Target]]` | Page redirect |
| `<noinclude>...</noinclude>` | Content hidden when transcluded |

### Fix a parser bug

1. Locate the issue in `src/TLUG/MediaWiki.hs`, `WikiLink.hs`, or
   `Parser.hs`.
2. Add a test in `src/TLUG/MediaWikiTest.hs` (follow existing
   `test_*` patterns).
3. Run `./Test --test-only` to verify.

### Fix a broken wiki link

Broken links render in red (`<font color=#CC2200>`). Check:
1. The target file exists in `wiki/` (case-insensitive match).
2. The link syntax is correct (spaces vs underscores, namespace).
3. If it's a transcluded template, check `TLUG.MediaWiki.transFileName`
   for how names map to files.

### Change site layout

Edit `template/main.html`. The `$body$` placeholder receives page
content. CSS is in `docroot/css/tlug4.css`.

### Add a new template for a page type

1. Create a `.html` file in `template/`.
2. Reference it in `app/SiteCompiler.hs` with `loadAndApplyTemplate`.
3. Rebuild and test.

## Important Constraints

- **No server-side runtime.** Everything must be pre-generated. Any
  interactivity needs client-side JS.
- **UTF-8 required.** Japanese characters are common. Build fails
  without a UTF-8 locale.
- **Transclusion recursion.** The parser resolves templates
  recursively. Be careful not to create loops.
- **Wiki filenames use colons.** `Meetings:2024:03` is a valid
  filename on Linux but may cause issues on some filesystems.
- **Extensionless wiki output.** Wiki files have no extension;
  `netlify.toml` sets `Content-Type: text/html` for `/wiki/*`.

  functions.

  browse `http://localhost:8000`.
